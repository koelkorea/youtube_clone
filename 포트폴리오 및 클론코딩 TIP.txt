1. 먼저 만들고자 하는 웹사이트의 틀부터 구조를 짜놔라
    -> 클론코딩이라면, 현재 클론할 웹 페이지나 앱의 HTML 영역을 기능 및 용도별로 구분해두어 역설계를 권장

2. 그렇게 짠 HTML 구조를 활용하여, 의미(Semantic)를 중점으로 HTML 태그를 설정하고, style을 위해 class명 또한 영역별로 명확하게 속성값을 주도록 하자
   (= Semantic 태그가 별거 아니어 보여도, 어떤게 있는지 잘 기억해두면 도움이 될듯)

3. HTML을 완성시키고 난 뒤, CSS를 작성할 때에는 자주 쓰이는 용도의 값은 '사용자 지정 CSS 속성'을 적극적으로 사용하여 의미까지 알 수 있게 하면 더 좋다
    -> 해당 페이지에 자주 쓰이는 색상이나, 폰트 크기 등을 용도별로 일종의 전역항목 역할을 하는 :root 의사클래스 안에 적어두면 좋음
       (단.. 이렇게 하면, CSS 전처리기, 후처리기를 통해 작성하는 것보다 압축단계가 없어서 용량이 커지는걸 감수해야 함)

        ex) :root {
                /* Color */
                --white-color: #fff;
                --black-color: #140a00;
                --blue-color: #045fd4;
                --red-color: #ff0000;
                --grey-dark-color: #909090;
                --grey-light-color: #e0e0e0;
            
                /* Size */
                --padding: 12px;
                --padding-small: 6px;
                --avatar-size: 36px;
            
                /* Font Size */
                --font-large: 18px;
                --font-medium: 14px;
                --font-small: 12px;
                --font-micro: 10px;
            }

4. 만약 본인이 생각하기에, 브라우저 환경에 따른 태그들의 기본설정 스타일 속성값이 있는게 변수처럼 느껴지면, * { }를 통해 전반적으로 신경쓰이는 속성값을 고정시켜라

    ex) * {
            padding: 0;
            margin: 0;
            box-sizing: border-box;
        }

7. 화면의 레이아웃을 알맞은 의도로 배치하고 싶을 때, flexbox를 쓰면 편한경우도 많지만, 이를 중첩해서 쓰면 나중에는 스파게티 코드가 될수 있으니.. 신중하게 구조를 짜자

8. 레이아웃을 짤 때, 자식요소의 크기나 영역설정을 위해 상대길이인 %를 쓰는건 아주 바람직하다..
    -> 하지만, 정말 이를 정밀하게 배치하고 싶으면, 적어도 %를 쓰는 해당 html태그 중 가장 선조에 해당하는 녀석의 직계 부모 태그의 길이는 절대단위로 주도록하자
